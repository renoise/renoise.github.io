Module xEffectColumn

A virtual representation of renoise.EffectColumn.

Unlike the renoise.EffectColumn, this one can be freely defined, without the need to have the line present somewhere in the actual song.

You can create an instance by feeding it a descriptive table in the constructor, or by letting the class read a specific effect-column (via the method do_read())

Note that all string-based values are automatically converted into numbers, and vice versa. Setting the note_string and then requesting the note_value will yield the same value, represented as string or number, respectively.
